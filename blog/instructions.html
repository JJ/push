How to write and submit a post to Push Blog

(Initially, the blog will focus on opening up the process and editorial and design decisions behind Push, as well as highlighting contributions and contributors to the journalâ€™s core files and scripts.)

  1. Fork the Push repository on GitHub and clone it to your computer. Create a new branch called post.
  2. Make a copy of the post starter file, _starter.md in the _posts directory, and name it YYYY-MM-DD-post-title.md (with the date and actual post title, of course).
  3. Fill in the information at the top of the file, and write up your post in Markdown.
  4. Track your progress with frequent Git commits.
  5. Push your work to your fork at GitHub.
  6. Submit by opening a pull request to the upstream Push repository by hitting the Pull Request button on GitHub on your fork of the Push repository.
  7. Make adjustments to your post, if necessary, and push to GitHub (your pull request will automatically update after you push your changes).

See the setup instructions and style guide for additional information and guidance. And if you need a question answered, just open an Issue on the Push repository.
